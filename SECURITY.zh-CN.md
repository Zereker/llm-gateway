# 安全策略

## 支持版本

在项目发布首个稳定版本之前，安全修复面向最新发布版本与默认分支。更早的提交以及
不再维护的部署示例不单独提供安全支持。

稳定版本发布后，本节会列出仍在安全维护期内的版本线和支持周期。

## 报告安全漏洞

请勿为疑似安全漏洞创建公开 Issue，也不要在公开讨论中提供真实 API Key、Prompt、
上游凭据、数据库内容或漏洞利用细节。

请通过 [GitHub Security Advisories](https://github.com/Zereker/llm-gateway/security/advisories/new)
私密报告漏洞。条件允许时请提供：

- 受影响的提交或发布版本；
- 复现问题所需的部署方式与配置；
- 使用合成凭据的最小复现步骤或 PoC；
- 预期影响与已知缓解措施。

维护者会尽快确认完整报告，通常不超过三个工作日。验证、修复、披露与发布周期取决于
严重程度和可复现性。公开披露前，请为修复保留合理时间。

## 安全边界

`llm-gateway` 会处理上游厂商凭据，也可能处理敏感的 Prompt 与响应内容。生产环境的
TLS 终止、Secret 管理、网络策略、持久化存储、数据库和消息系统访问控制、日志保留策略，
以及内容日志开关均由部署方负责。示例和 Quickstart 凭据只能用于开发环境。
